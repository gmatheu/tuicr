# `--in-repo` CLI flag for persistence mode opt-in

## Request

Add a CLI flag that opts into in-repo persistence. The flag needed to be parsed by the existing CLI system, propagated into final app state/config resolution, and made observable in tests. The change also needed to stay aligned with the persistence-mode plan so the app can resolve local vs repo persistence from startup inputs.

## Response / Implementation

- Kept `CliArgs` carrying `in_repo: bool` in `src/theme/mod.rs` and ensured the parser recognizes `--in-repo`.
- Threaded the flag into startup resolution so `main` can pass it into persistence-mode selection and app construction.
- Cleaned up CLI help text and README usage docs so the flag is visible to users.
- Kept parsing coverage focused on the opt-in behavior (`--in-repo` => `true`, omitted => `false`).

## Notes for future work

- Persistence now has a clear startup toggle surface: config can still decide the default mode, while the CLI flag overrides it when present.
