Task summary: Hg username retrieval feature

- Request: Add Mercurial username retrieval helper for Hg backend, using `hg config ui.username` with a fallback to "anonymous".
- Context: Location: src/vcs/hg/mod.rs. Implement a get_current_username() that runs the Hg command and parses its stdout; on error or empty result, fallback to anonymous.
- Outcome details:
  - Code changes: Modified src/vcs/hg/mod.rs to add a get_current_username() method on HgBackend (within impl HgBackend) that executes `hg config ui.username` via Command, returns trimmed output if present, otherwise returns "anonymous".
  - Tests: Added unit tests under Hg backend tests to cover both configured username and fallback behavior.
    - test_hg_get_current_username_configured: Configures a per-repo username in .hg/hgrc and asserts the method returns that username.
    - test_hg_get_current_username_fallback: Ensures no username configured and asserts the method returns "anonymous".
  - Verification approach: Run `cargo test`. Tests are designed to skip if Mercurial is not available on the system. If available, tests create temporary Hg repos and validate behavior.
- Notes on scope:
  - This change is isolated to Hg (Mercurial) backend only; no changes to Git or JJ backends.
  - The new API is a method on HgBackend used by internal code paths; the tests leverage existing test helpers to create repos.
- Rationale: Centralizes environment config access in Hg backend and ensures deterministic fallback behavior for environments lacking username config.
